# Итоговый отчёт по анализу величин на графиках

Документ объединяет результаты проверки согласованности данных с графиков ЯМР и блоков вычислений (calculations.pdf / calculations_verified.json), а также выявленные неточности и расхождения.

---

## 1. Источники данных

| Источник | Описание | Откуда берутся K, Pr и производные |
|----------|----------|-------------------------------------|
| **Графики** | `graphics_merged.json`: заголовки и метрики по страницам графиков (без/с покрытием). | K1, K2, Pr1, Pr2 — из `structured_metrics` заголовка (crystallinity_index, proton_density). G1 по графику: **G1_merged = K2/K1 + Pr2/Pr1**. |
| **Блок вычислений** | `calculations_verified.json`: 78 блоков (страницы 1–39 по 2 графика). В каждом блоке: K1, K2, L1, L2, P1, P2, R1, R2, G1, G2, G, D. | G1 в блоке (G1_PDF) — результат расчёта по той же формуле G1 = K2/K1 + Pr2/Pr1 по числам, приведённым в PDF. G2 = L2/L1 + R2/R1; G = G2 − G1; в PDF для D используется **D = G1/G**. |

По методологии (п. 6.4) K и Pr получают из разложения ССИ (короткая/длинная компоненты) и массы навески; в заголовке графика они уже представлены как индекс кристалличности и протонная плотность. То есть **числа с графиков (заголовки) и числа в таблице расчётов по идее должны относиться к одним и тем же измерениям**.

---

## 2. Должны ли вычисления напрямую следовать из графиков?

Да. Если блок вычислений соответствует конкретному графику (пара «без покрытия» / «с покрытием»), то:

- Входные величины блока (K1, K2, P1/Pr1, P2/Pr2) должны совпадать с теми, что указаны на графике (в заголовке или в сопроводительной таблице).
- Тогда **промежуточная величина G1** = K2/K1 + Pr2/Pr1, рассчитанная по данным графика, и **G1 из блока вычислений** должны совпадать с точностью до округления.

**Любое систематическое расхождение между G1 по графику и G1 из блока вычислений является критичным**: оно означает либо что в расчёты подставлены не те данные, что на графике, либо ошибки в расчётах/округлении, либо что выводы в документе не опираются на те же первичные данные, что представлены на графиках.

---

## 3. Расхождение G между блоком графиков и блоком вычислений (критичная неточность)

### 3.1. Сопоставление

- Для **77 графиков** (страницы 1–39) блок вычислений однозначно определяется по индексу (страница + номер графика). Для этих строк сравниваются:
  - **G1_merged** — G1, рассчитанный по данным из заголовков графиков (merged);
  - **G1_PDF** — G1 из соответствующего блока calculations.

### 3.2. Результаты сравнения

| Показатель | Значение |
|------------|----------|
| Графиков с блоком по индексу | 77 |
| Макс. \|G1_merged − G1_PDF\| | **63.15** |
| Средняя \|G1_merged − G1_PDF\| | **5.65** |
| Средняя относительная разница | **~31%** |
| Строк с выбросом (\|ΔG1\| > 1 или > 10%) | **66 из 77** |

Примеры (страница, график, G1 по графику, G1 из PDF, разница):

- стр. 2 g2: G1_merged = 142.13, G1_PDF = 78.98, Δ ≈ 63.1;
- стр. 4 g1: G1_merged = 188.35, G1_PDF = 156.04, Δ ≈ 32.3;
- стр. 7 g2: G1_merged = 32.97, G1_PDF = 57.16, Δ ≈ −24.2.

Исключением является, например, стр. 1 g1: G1_merged ≈ 6.9324, G1_PDF ≈ 6.9323 — совпадение в пределах округления.

### 3.3. Пример проверки по формулам: страница 3, график 2

**Формулы:** G1 = K2/K1 + P2/P1,  G2 = L2/L1 + R2/R1  (P — протонная плотность).

**Данные с графика (стр. 3, график 2):**

| Параметр | Значение |
|----------|----------|
| Индекс кристалличности 1 (K1) | 0.80772 |
| Протонная плотность 1 (Pr1) | 0.33274 |
| Индекс кристалличности 2 (K2) | 0.79862 |
| Протонная плотность 2 (Pr2) | 2.10588 |

**Данные из блока calculations (тот же образец):**

| Параметр | Значение |
|----------|----------|
| K1 | 0.80772 |
| K2 | 0.79560 |
| L1 | 0.70227 |
| L2 | 0.77173 |
| P1 | 0.33274 |
| P2 | 2.44711 |
| R1 | 0.32139 |
| R2 | 2.37370 |

**Расчёт G1 по данным графика:**

G1_график = K2/K1 + Pr2/Pr1 = 0.79862/0.80772 + 2.10588/0.33274 = 0.98872 + 6.3264 ≈ **7.315**

**Расчёт G1 по данным блока calculations:**

G1_расчёт = K2/K1 + P2/P1 = 0.79560/0.80772 + 2.44711/0.33274 = 0.98499 + 7.3532 ≈ **8.338**

**Расчёт G2 по данным блока (на графике L и R не приведены):**

G2_расчёт = L2/L1 + R2/R1 = 0.77173/0.70227 + 2.37370/0.32139 = 1.0989 + 7.3840 ≈ **8.483**

**Сравнение входных величин графика и calculations:**

| Величина | График | Calculations | Совпадение |
|----------|--------|--------------|------------|
| K1 | 0.80772 | 0.80772 | ✓ |
| K2 | 0.79862 | 0.79560 | **разные** |
| P1/Pr1 | 0.33274 | 0.33274 | ✓ |
| P2/Pr2 | 2.10588 | 2.44711 | **разные** |

**Вывод по примеру (стр. 3, гр. 2):** G1 не совпадает (7.315 vs 8.338), потому что в блоке calculations подставлены другие K2 и P2, чем на графике. Формула одна и та же; расхождение вызвано разными входными данными.

### 3.4. Вывод

**Расхождение G (G1) между данными с графиков и блоком вычислений является критичным.** В большинстве случаев G1, рассчитанный по заголовкам графиков, не совпадает с G1 из блока расчётов. Это указывает на то, что:

- либо в блоке вычислений использованы **не те входные данные**, что отражены на графиках (другой источник или другая интерпретация);
- либо имеются **ошибки в подстановке или расчёте** в документе расчётов;
- либо **порядок сопоставления** «график ↔ блок» в части случаев неверен (что не отменяет массовость расхождений).

В любом из этих вариантов утверждение о том, что «вычисления напрямую следуют из графиков», **не выполняется** для большинства образцов. Различие промежуточных величин (G1) в данной ситуации действительно критично для обоснованности выводов по давности.

## 4. Другие выявленные неточности и особенности

### 4.1. Формула расчёта давности D: методология vs calculations.pdf

| Источник | Формула D |
|----------|-----------|
| Методология (п. 6.6) | **D = (G2 − 2) / G**, где G = G2 − G1 (калибровочная прямая с вершиной в точке G = 2 при D = 0). |
| calculations.pdf | **D = G1 / G** (во всех 102 проверенных строках с блоком D_PDF совпадает с G1/G с точностью &lt; 0.5). |

Это **разные формулы**. При типичных значениях G1, G2 формула из calculations даёт систематически бóльшую давность (например, ~57 мес. вместо ~42 мес. по методологии для блока 1). Итог: в расчётах используется формула, **не совпадающая с описанной в методике**.

### 4.2. Постоянство отношений K2/K1 и L2/L1 (deep_analysis_report)

По всем блокам calculations:

- **K2/K1**: среднее ≈ 0.985, СКО ≈ 0.000004;
- **L2/L1**: среднее ≈ 1.099, СКО ≈ 0.000004.

При независимых измерениях разных образцов и реквизитов ожидался бы заметный разброс. Практически постоянные отношения согласуются с гипотезой о **подборе входных величин** под целевую зависимость или заданные константы.

### 4.3. Концентрация заявленных значений D (подгонка под целевое D)

- В calculations заявленные D сосредоточены в **двух узких областях**: около **57.42 мес** (≈69 блоков) и около **56.24–56.26 мес** (≈24 блока). Уникальных значений D всего 5 из 78.
- Отношение **G2/G1** по блокам практически постоянно (среднее ≈ 1.0175, СКО ≈ 0.00017), что при фиксированном целевом D = G1/G даёт жёсткую связь между G1 и G2 и согласуется с **подгонкой под заданное D**.

Риск: признаки возможной **подгонки экспертного вывода под заранее заданный результат** давности.

### 4.4. Страницы 40+ графиков: расхождение полей K и L

- В calculations_verified.json только **78 блоков** (страницы 1–39). Для страниц 40+ блока по индексу нет.
- В merged для страниц 40+ в полях, соответствующих K, нередко попадают значения **L** из PDF (диапазон K смещён вниз относительно страниц 1–39). Для части графиков блок подбирался по совпадению L1, L2 («по L1,L2»); для 25 графиков блок так и не найден.

Это указывает на **несогласованность структуры данных** (номенклатура/источник полей) между разделами документа для страниц 40+.

### 4.5. Ранний анализ LLM (graphics_llm_summary)

При сопоставлении по (K1, K2) для 51 пары блок–график **ни в одном случае** G1 по графикам не совпадал с G1 из PDF, и **ни в одном случае** D по формулам не совпадала с D в PDF в пределах ±0.5 мес. Это согласуется с выводами о расхождении G и о том, что расчёты в документе не воспроизводятся по данным с графиков.

## 6. Выводы

1. **Связь «график → вычисления» не подтверждается.** Для большинства графиков (страницы 1–39) величина G1, рассчитанная по данным из заголовков графиков, существенно отличается от G1 в соответствующем блоке вычислений. **Различие промежуточных величин (G) в данном контексте критично**: оно указывает на то, что вычисления в документе либо не опираются на те же данные, что представлены на графиках, либо содержат ошибки подстановки/расчёта.

2. **Формула давности в calculations не соответствует методологии.** В документе используется D = G1/G вместо заданной в методике D = (G2−2)/G, что даёт систематически завышенные значения давности.

3. **Статистика входных и выходных величин** (постоянство K2/K1, L2/L1, концентрация D, постоянство G2/G1) согласуется с гипотезой о подгонке входных данных и результата под заданную давность, а не с независимыми измерениями по методике.

4. Для формирования окончательных экспертных выводов целесообразно: (а) установить фактический источник чисел в блоке вычислений (таблицы, заголовки графиков, ручной ввод); (б) проверить соответствие формулы D методике; (в) при необходимости провести перерасчёт G1 и D по единообразным правилам по данным с графиков.
